# CllineAI

## Автосборка и автопубликация Windows-сборки

В репозиторий добавлен GitHub Actions workflow `.github/workflows/build-and-release-exe.yml`.

Что происходит автоматически:
- при каждом `push` в `main/master` (и при ручном запуске) GitHub собирает проект в режиме `Release`;
- формируется дистрибутив `CleanAI-windows-x64.zip`, который содержит `CleanAI.exe` и необходимые runtime-зависимости (`.dll`);
- архив загружается в workflow artifacts;
- этот же архив публикуется/обновляется в GitHub Release с тегом `latest`.

То есть после каждого обновления кода вы получаете актуальную portable-сборку для запуска на «чистой» Windows-машине без ручной докомплектации библиотек.


## Автовосстановление при падении CI

Добавлен workflow `.github/workflows/auto-recover-build.yml` с многошаговым сценарием восстановления:
- если сборка `Build and release EXE` упала на первой попытке — автоматически перезапускаются **только упавшие job'ы**;
- если вторая попытка всё ещё красная — автоматически перезапускается **весь workflow целиком**;
- если после трёх попыток ошибка сохраняется — автоматически создаётся issue с деталями run и чеклистом для ручного разбора;
- если один из автоматических ретраев всё-таки починил сборку — связанный incident-issue автоматически закрывается с комментарием об успешном восстановлении.

Это уже не просто единичный ретрай, а полноценная «автоаварийка», которая снимает с вас рутину при флаках и временных сбоях CI.
